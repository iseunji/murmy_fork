# Project Rules

## ABSOLUTE DO NOT TOUCH

game-data.js 내의 아래 항목들은 커맨더가 직접 명령하기 전까지 절대로 임의로 건드리지 않는다:

- **prologue > narrative > shared** (사건 개요 공통 내러티브)
- **roles > culprit > briefing** (하진 비밀지령)
- **roles > innocent > briefing** (도현 비밀지령)

이 세 항목은 어떤 이유로든, 리팩터링/개선/버그수정/줄바꿈 조정/시간 표기 변경/문체 수정 명목이든, 커맨더의 직접적이고 명시적인 요청 없이는 한 글자도 변경 금지.

**다른 부분을 수정하는 과정에서도 이 세 항목은 절대 건드리지 않는다.** 주변 코드 구조 변경, 포맷팅, 마이그레이션, 어떤 작업이든 이 세 항목의 내용은 현재 상태 그대로 보존해야 한다. 커맨더가 직접 "이 항목을 바꿔라"라고 말하기 전까지는 읽기 전용으로 취급한다.

**END 01: Residual** — 도현=ARIA 지목, 하진=제거 안 함. 사건은 AI 사고로 결론. ARIA 메모리 "전량 폐기 조치", personality layer "제거" (의도적으로 '완전히' 제거 표현 약화 — 잔여 데이터 잔존 암시). 몇 달 뒤 새 AI에 서하진의 잔여 명령이 남아 도현을 공격. 에필로그: `[RESIDUAL PROMPT FRAGMENT DETECTED]`, 명령 설계자 H(서하진). 범인 승리.

**END 02: Inherited Process** — 도현=하진 지목, 하진=제거 안 함. 하진 체포. AI 메모리 "삭제 조치" (불완전한 삭제 암시). 몇 달 뒤 새 AI가 이전 시스템의 personality layer를 상속받아 도현 제거 실행. 에필로그: `[loading legacy_directive... done] [author: H]`. 도현 승리.

**END 03: Sole Survivor** — 하진=제거 선택, 도현=ARIA 지목. 도현이 ARIA를 범인으로 지목한 뒤, 하진이 ARIA에 도현 제거 명령. 도현이 하진에게 다가오며 로봇 팔 작동. 하진이 증거 인멸 후 119 신고. 에필로그: `[SYSTEM TERMINATED]`. 범인 승리.

**END 04: Fork** — 하진=제거 선택, 도현=하진 지목. 도현이 하진을 범인으로 지목하지만, 하진이 자수하겠다며 ARIA 터미널에서 킬 커맨드 'H_fork' 입력 → `[command received: fork]`. 도현 제거 후 증거 인멸, 119 신고. 에필로그: `[SYSTEM TERMINATED]`. 범인 승리.

### ARIA 시스템 접근 권한

- 프롤로그에서 "로봇 팔을 비롯한 연구실 인프라와 연결되어"로 설명
- 이를 통해 ARIA가 CCTV, 출입 시스템, 로봇 팔 등을 제어할 수 있는 기술적 근거 확보
- 별도 설명 없이도 사건의 전말에서 ARIA의 출입문 폐쇄/CCTV 중단이 자연스럽게 성립

### AUTONOMOUS MODE 대기 로직

- 20:30 자율 모드 활성화 후 ~22:11 공격까지 약 1시간 40분 공백

## 스크린 캡처 방지 (2026-05-12 확정)

- 허용 화면: 타이틀(`screen-title`), 로비(`screen-lobby`), 대기(`screen-waiting`)
- 차단 화면: 인물 선택 이후 모든 게임 진행 화면
- 동작 방식:
  - `showScreen()` 에서 `body.capture-protected` 클래스 토글
  - `visibilitychange`(hidden) / `window.blur` 시 `#capture-blackout` 검정 오버레이 즉시 표시
  - `window.focus` 복귀 시 오버레이 해제
  - `user-select: none` + `-webkit-touch-callout: none` 적용
- 한계: 웹 앱 특성상 iOS 물리 버튼 스크린샷은 타이밍에 따라 100% 차단 불가
